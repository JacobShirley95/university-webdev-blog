Week 4: Website upgrade

Jacob Shirley

October 24th, 2016

This week I created a style for my journal, tutorial and contact pages. To begin with I started using a website called JSFiddle, which is an online platform for building HTML/CSS/JavaScript demos[5]. I only had a vague idea for what a style I wanted and that was minimalistic. I started by adding a margin to the wrapper so that the posts would appear centered.

I then started working on the style of the posts. I wanted them to be easily readable but also making the page look neat and without clutter. I finally settled with a thin box shadow generated using the CSSmatic box shadow generator[6]. An early prototype can be seen here.

Next I found some fonts on the Google Fonts[7] website and styled the h1..h6 tags to use them.

Once the style of the posts was done I started work on the page header (title and tagline) and the navigation bar. For the header all I did was centre the text (text-align: center). For the navigation bar a little more work was needed to centre it: I used the flex layout system so that each navigation element would grow or shrink to fill the remaining space. This has another advantage of being easily customisable: if I want to add a new navigation item, no other CSS code will be needed to make it fit equally. At this point I styled the page links so that they aren't the ugly default colours, and also used a small amount of JavaScript, using jQuery[8], to add a red border to a nav item when hovered over.

Finally after validating the HTML I decided I needed a quicker way of writing posts since I won't be using a content management system backend. The solution I came up with was to build a page to generate a post[9]. All it does is insert information from a form into a string and outputs it into the output text area. I can then copy and paste this into my journal. I may keep adding to it but at the moment it uses no CSS so it's not at all user friendly.

  • Started styling.
  • Added a "tag" class to easily display < and > symbols around tags.
  • Added box shadow to posts.
  • Found fonts and added them to the h1..h6 tags.
  • Styled and centered the page header using text-align: center.
  • Styled and centered the navigation bar using flex.
  • Removed default styling from links and added a hover state to nav items.
  • Created a post generator.
  • Styled blockquote and table.
  • Added normalize.css.
  • Validated my CSS using the W3C CSS Validator[11].

Week 5: CSS restructuring, removed nav JavaScript, table of contents, new CSS

Jacob Shirley

October 28th, 2016

This week I did some CSS updating and restructuring: moved the basic CSS styles, such as h1..h6, to the top of the document, and added h5..h6 to the h4 styles; I renamed the selector .journal-title to #page-title, as well as making it an id; I removed the div containing the page title and tag line and made the surrounding header perform that function instead, I updated the line-height (to 1.5) property of the page to make it more readable, and I styled the HTML logo (float:right) and the img tag in the CSS document rather than inline-styling. I also put in some comment headings for anyone reading it.

At this point I realised there was no need to use JavaScript for the border on the navigation links: it could all be done through CSS (a:hover). I therefore removed the .active CSS class which was originally used by JavaScript to be either added to the navigation list items (li) or removed when a link was hovered.

What I also needed to add was a table of contents. This is useful because it makes the journal much more user friendly as it can easily be navigated, and I get the chance to use the aside tag and in-page links in a real context. I also wanted it to be sticky: when the user had scrolled beyond the "top" coordinate of the table of contents, it should "stick" to the browser viewport so it's always visible. I managed to achieve this using JavaScript as there is currently no viable way to do this through CSS alone. I therefore made a small JavaScript class called StickyMenu to make this code reusable for future projects (it's in script.js[12]). However, I didn't want this to happen when viewed on screens smaller than 1000px (i.e. mobiles). For this I used a small JavaScript library called enquire.js[13] which allows JavaScript code to be executed when a CSS media query is activated or deactivated; in this case to enable the sticky menu when screen size is above 1000px and disable it when below 1000px. I also applied the hover styles of the table of contents links (class .link-item) to the navigation links.

After reading the tutorial page, I decided to test some CSS I hadn't used before: text-indent, :first-letter, and clearing text around images with clear. Below is a demo of each of these:

Week 6: Added section headings, cleaned up tags, moved nav bar to sidebar, made page responsive

Jacob Shirley

October 30th, 2016

According to the W3C HTML validator, every section element requires a header. However, for my main posts section I couldn't create a header that looked good, so I added a new class "hidden" specifically for elements that are needed by screen readers but look out of place when on a normal screen, when they are just hidden.

I also discovered a cleaner solution to styling tag elements, i.e. writing divs in plain text without it being interpreted as HTML. I used an attribute selector (*[data-tag])[15], where any element with a data-tag attribute is styled with tag properties (bold, with < and > before and after it). This looks much neater from an HTML point-of-view.

I decided to combine the navigation list with the sidebar. This made sense for a couple of reasons: it adds more continuity by grouping related parts of the page together, and because they use the same styles; it also means that, since the layout of the tutorial page is the same (the margins on either side of the main content is the same), there's no wasted space around the tutorial content.

It was also about time to make the site responsive. I had built the site for desktop first, which is probably not the best way to do it these days, so I wanted the mobile site to look mostly like the desktop version. All of this is done using media queries[16]. For a start, I thought there would be too much extra space around the posts on screen sizes less than about 1500px but more than 1000px, so I removed most (down to 1%) of the right padding on the #wrapper. I then defined the mobile site as less than 1000px; I removed the margins of the #wrapper and made the sidebar have a relative, instead of absolute, position using position: relative, so that it doesn't move.

Summary:

  • Added hidden headings to section tags.
  • Cleaned up the HTML of tag elements by using attribute selectors.
  • Moved navigation list to the sidebar.
  • Made the page responsive using media queries.

Week 8: Hamburger menu, cleaning up CSS, using the nav tag

Jacob Shirley

December 16th, 2016

For this week's task I decided to make the table of contents use a hamburger icon on small screen using only CSS. For the icon itself I looked into ways to make it generated by CSS, so that no extra images would be needed to be loaded. I ended up using the pseudo gradient CSS code on CSS Tricks[17] for the task.

With the icon ready I then needed to set the initial table of contents height so that only the title was showing and not the list items. I set it to 2em, which is roughly the size of the h1 tag. Finally I needed it to expand to it's normal size when hovered over, so I added the pseudo class[18] :hover to the .table-of-contents style. When hovered it would set the height property to auto.

This all worked well on Desktop and Android devices, but on iOS mobiles there was a bug where the :hover pseudo-class won't activate on a touch event. To fix this I had to add an onclick="void(0)" to the table of contents section, so that iOS sees it as a button element. There is still a bug, however, where the table of contents won't close on iOS despite clicking away from it. Given more time I may be able to fix this but since I don't have an iOS mobile, it's very hard for me to test/bugfix at home.

Other changes I made this week are removing old/unused CSS and wrapping the navigation bars (Navigation and Table of contents) with the nav element.

Summary:

  • Started the hamburger menu for small screen devices (i.e. mobiles).
  • Removed old/unused CSS code.
  • Wrapped navigation bars with nav elements.
  • Validated CSS on the W3C CSS validator.
  • Validated HTML on the W3C HTML validator.

Week 9: Contact form, TOC scrollbar, feedback

Jacob Shirley

January 6th, 2017

For week 9 I started working on the contact form. I used the HTML on the week 9 course tutorial page[19] as a basic outline for the form. I then started styling it in the CSS document: I removed the border on the fieldset tags, made the fonts look better on label and legend, and disabled horizontal resizing of the textarea using resize: vertical, among other things.

I solved a problem where some of the table of contents would not be visible when the browser viewport was too small, and there was no scrollbar. To fix this, I found a stack overflow solution[20] and I set the top and bottom CSS properties on the sidebar to 0, and set overflow-y to auto. Now when the viewport is too small a scrollbar appears.

Finally I asked a few family members and friends to give me feedback on the site. The main response was that I could have used some more colour to make it look more interesting, but they said they liked the sidebar and how it stays visible when scrolling.

Summary:

  • Made and styled the contact form.
  • Fixed a table of contents scrollbar bug.
  • Got feedback.
  • Validated HTML and CSS.
